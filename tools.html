Procedural 3D Texture Generator (Unity Editor / C#)

I developed a custom editor wizard tool within Unity to instantly generate volumetric 3D texture assets powered by procedural noise math.

Three-dimensional volume preview of the generated noise inside the Inspector

Objective & Integration

The main goal of this tool was to remove the reliance on external third-party software for creating complex volumetric noise maps, providing a lightweight, in-engine solution to experiment with different resolutions and frequencies on the fly.

  • Unity Editor: Built as a custom EditorWindow utility seamlessly integrated into the editor layout.
  • Unity Mathematics: Leveraged the mathematical library to generate optimized 3D Simplex noise via the noise.snoise() function.

Editor Window Configuration

The wizard is accessible from the top menu via Tools → Générer Texture 3D Noise. The user configures the generation through three primary parameters:

Parameter Description
Size The cubic resolution of the texture across all three axes (X, Y, Z). e.g., 32³, 64³, 128³.
Frequency Controls the scale and tiling frequency of the noise pattern in 3D space.
File Name The target name for the final `.asset` file saved within the project directory.

1. Initializing the Texture3D Container

The script instantiates an empty Texture3D container using the user-defined size. It uses the RGBA32 color format. To ensure seamless looping and clean lookups, the filtering is set to Bilinear and the WrapMode is forced to Repeat.

Texture3D texture = new Texture3D(size, size, size, TextureFormat.RGBA32, false);
texture.wrapMode = TextureWrapMode.Repeat;
texture.filterMode = FilterMode.Bilinear;

2. Mathematical Simplex Sampling

Using three nested loops (X, Y, Z), the script calculates the normalized local position of every single voxel. Since snoise() outputs values in a -1 to 1 range, a standard mathematical remapping is applied to bring it into a [0, 1] range fit for color grayscale maps.

float3 pos = new float3(x, y, z) / (float)size * frequency;
float noiseValue = (noise.snoise(pos) + 1f) * 0.5f;

colors[index] = new Color(noiseValue, noiseValue, noiseValue, 1f);

3. Data Writing & UX Selection Focus

Once the pixel color array is completely populated, the texture changes are finalized via Apply(). The tool writes the native asset into the project folder using the AssetDatabase API.

For optimal user convenience, the script sets the editor's active selection directly to the newly created asset so it pops up instantly in the Inspector window.

texture.SetPixels(colors);
texture.Apply();

AssetDatabase.CreateAsset(texture, path);
AssetDatabase.SaveAssets();

Selection.activeObject = texture;

One-click generation process with automatic asset focus and highlighted inspector selection

Technical Breakdown & Future Outlooks

This tool successfully created an immediate iteration loop for generating noise variants. However, because the computational logic currently runs entirely on the CPU, generation times can scale drastically up when computing higher resolution grids (e.g., 256³).

Project Note: As the pipeline pivoted away from using this specific resource type, further development on the tool was halted. Future steps would have involved shifting the generation mathematical load onto the GPU via Compute Shaders, alongside implementing Worley and Fractal Brownian Motion (FBM) noise algorithms.

Modular Map & Level Generator (Unity Editor / C#)

An editor-integrated pipeline tool developed to automatically instantiate and align modular environments (rooms, corridors) based on manual layout anchor points.

Real-time modular environment instantiation directly in the Scene View

Objective & Workflow

Built to accelerate grayboxing, prototyping, and layout assembly phases. The system allows designers to map out pathways or room outlines via anchor nodes, instantly managing the correct placement and orientation of walls, corner trims, floors, and ceilings.

Tool Architecture & Editor UI

To strictly separate core generation mathematics from interface rendering, the pipeline is decoupled into two primary C# components:

  • MapGeneratorEditor: A custom Inspector implementation providing an ergonomic layout window with instant controls.
  • MapGenerator: The underlying modular processing engine handling geometry routines and asset pooling.
Inspector Button Internal Operation
Generate Map Triggers the full layout generation pass based on active anchor data.
Clear Map Safely purges and resets previous procedural instances to ensure clean scene state.

Path Logic & Generation Routine

Environmental data structures are declared via structural vectors. Each custom path segment handles data collections such as an array of anchor nodes, section branding names, open/closed loop Boolean parameters, and tiling flags for ceilings and floors.

1. Wall Fitting & Corner Tracking

The system iterates along anchor arrays to compute segment directions and distances. Wall prefabs are automatically spawned at consistent steps along the segment vector, inheriting matching rotational data.

Where two distinct segments meet, corner angle calculations trigger specific trim prefab variants to prevent ugly physical overlaps.

Vector3 direction = (targetNode - startNode).normalized;
Quaternion wallRotation = Quaternion.LookRotation(direction);
GameObject wall = Instantiate(wallPrefab, spawnPos, wallRotation);

2. Smart Flood Fill Tiling

Placing thousands of floor and ceiling units by hand is counterproductive. To bypass this, an initial bounding grid array is laid down following structural boundaries.

Then, a customized Flood Fill propagation algorithm automatically crawls empty spaces, scanning adjacent neighbor entries to cleanly patch inner rooms without tile duplication.

if (!tileRegistry.Contains(targetPos) && CheckNeighbors(targetPos)) 
{
    SpawnGridElement(targetPos);
    tileRegistry.Add(targetPos);
}

3. Hierarchy Routing & Bounds Check

For closed layouts, geometric poly-checks ensure tiles never leak outside designated perimeters. Instantiated assets are categorized automatically inside a strict nested structure to ensure easy debugging.

Transform roomParent = new GameObject(roomName).transform;
roomParent.SetParent(mapRootContainer);
wallInstance.transform.SetParent(roomParent);

Production Impact & Key Takeaways

The implementation of this tool was highly successful, saving substantial production time by bypassing the repetitive task of manually positioning over 3,000 modular tiles and hundreds of walls. Whenever structural parameters evolved, regenerating the entire base infrastructure only required adjusting a few anchor positions.

  • Design Constraints: The tool operated under specific design limits—anchor distances required increments of 2 units to strictly match the 2-meter physical dimensions of the core wall assets.
  • Human Intervention: The framework was purposefully built as an infrastructure layout generator rather than an absolute automated solution. Level designers still stepped in manually to populate the environments with props, interactive doors, and fix minor corner overlaps where custom shapes deviated from the 2-meter tile metric.
